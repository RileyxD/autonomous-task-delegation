# Security Policy

## Supported versions
Only the latest tagged release is supported with security fixes.

## Reporting a vulnerability
Email the maintainer privately with:
1. A clear description of the issue.
2. Reproduction steps.
3. Potential impact.

Please do not open a public issue for unpatched vulnerabilities.

## Security guidance for users
1. Keep agent permissions conservative where possible.
2. Review delegated code changes before merging.
3. Avoid passing secrets in plain text prompts.
4. Rotate credentials if prompt/output logs contain sensitive data.
